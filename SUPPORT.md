# Support

Use GitHub Issues for public-safe bugs, docs gaps, manifest questions, and demo
proposals.

## New here?

If you are just getting oriented, the [Tour](docs/tour.md) is the fastest way
to see the repo end to end. The [FAQ](FAQ.md) covers the most common questions
about what the repo does and how it fits into agent workflows. The
[Glossary](docs/glossary.md) bridges cryo-EM vocabulary and CryoCore
orchestration terms. If a question is still open after those, open an issue
with the labels suggested in the templates.

## In Scope

- Public-safe validator bugs.
- Documentation gaps.
- Public accession demo issues.
- Schema, manifest, or skill-pack questions.
- Reproducible release-gate failures.

## Out Of Scope

- Clinical, therapeutic, legal, or licensing advice.
- Private data review in public issues.
- Debugging proprietary or license-gated software access.
- Operating paid provider runs for users.
- Interpreting biological mechanism claims beyond the evidence in a dossier.

Support is best effort for the pre-alpha public repo.

Do not post:

- credentials, tokens, signed URLs, provider IDs, billing records, or raw logs
- private biological data, unpublished structures, private maps, sequences, or
  model weights
- raw cryo-EM movies, particle stacks, half-maps, masks, or heavy generated
  artifacts
- license files, private installer URLs, or third-party access credentials

Use the private vulnerability reporting path in `SECURITY.md` for sensitive
security or private-data concerns.
